# gitp ロードマップ

最終更新: 2026-04-02

## リブート方針

gitp.sh の原型に立ち返り、Rust 版をシンプルに再構築する。
機能を足すのではなく削ぎ落とす。

### リネーム候補

公開時の crate 名は **`gitpp`**（crates.io 空き確認済み 2026-04-02）。
git + personal + parallel。ローカルのバイナリ名・リポ名の変更は公開時に判断。

## 現状の課題

### 削除すべきもの

| 項目 | 理由 |
|---|---|
| `config` / `config user` コマンド | clone/pull/push で自動適用される。独立コマンドは不要 |
| `serial` オプション | 並列のみに統一。現状も未実装 |
| インタラクティブモードの config user 補完候補 | コマンド廃止に伴い |

### 修正すべきもの

| 項目 | 内容 |
|---|---|
| macOS 判定 | "unknown OS" ではなく macOS として認識する |
| デバッグ println | OS 判定時の毎回出力を削除 |
| YAML パースエラー | 空設定で続行せず、エラーを出して終了すべき |
| git_status() | 未使用。呼ばれていないなら削除 |
| 設定ファイル名 | `gitp_setting.yaml` → `gitp.yaml` にリネーム |

### 新規実装

| 項目 | 内容 |
|---|---|
| TUI スクロール | 100リポ対応。一覧を縦スクロール可能にする |
| TUI 詳細ペイン | Enter で右ペインに選択リポの git 出力をリアルタイム表示 |
| TUI 終了後サマリー | 失敗リポの git 出力を stdout に流す |
| TUI キー操作 | j/k スクロール、Enter 詳細展開、Esc 閉じる |

## 移行ステップ

1. **現コードの修正** — 削除・修正すべきものを処理
2. **TUI 改修** — スクロール、詳細ペイン、終了後サマリー
3. **動作確認** — gitp.sh と並行して pull を実行し、結果を比較
4. **gitp.sh 廃止** — `cargo install` で Rust 版に完全移行
